# CI/CD Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up three GitHub Actions workflows — deploy frontend to GCS, deploy FastAPI backend to Cloud Run, deploy graph-gen job image to Artifact Registry — all authenticating to GCP via Workload Identity Federation (no service account JSON keys).

**Architecture:** Each workflow is path-scoped so only the affected component redeploys on push. Authentication uses the `google-github-actions/auth` action with WIF. Frontend deploy runs `next build` then `gsutil rsync`. Backend and job deploys build Docker images, push to Artifact Registry, then trigger Cloud Run update.

**Tech Stack:** GitHub Actions, google-github-actions/auth@v2, google-github-actions/setup-gcloud@v2, Docker

**Prerequisites:**
- infra plan complete (WIF pool + provider created, service accounts exist)
- `terraform output wif_provider` and `terraform output github_actions_sa_email` values noted
- backend and graph-gen job plans complete (Dockerfiles exist)
- frontend plan complete (`next build` works)

## Task 2: Frontend Deploy Workflow

- [ ] **Step 1: Create `.github/workflows/deploy-frontend.yml`**

```yaml
name: Deploy Frontend

on:
  push:
    branches: [main]
    paths:
      - "frontend/**"
      - ".github/workflows/deploy-frontend.yml"

permissions:
  contents: read
  id-token: write   # Required for WIF

jobs:
  deploy:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: frontend

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: "22"
          cache: "npm"
          cache-dependency-path: frontend/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build
        env:
          # Empty string → relative URLs (/api/graph). CDN routes /api/* to Cloud Run.
          NEXT_PUBLIC_API_URL: ""

      - name: Authenticate to GCP
        uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: ${{ vars.WIF_PROVIDER }}
          service_account: ${{ vars.WIF_SERVICE_ACCOUNT }}

      - uses: google-github-actions/setup-gcloud@v2

      - name: Deploy to GCS
        run: |
          gsutil -m rsync -r -d out/ gs://${{ vars.GCS_BUCKET }}/

      - name: Invalidate CDN cache
        run: |
          gcloud compute url-maps invalidate-cdn-cache ${{ vars.CDN_URL_MAP }} \
            --path "/*" \
            --async \
            --global
```

- [ ] **Step 2: Commit and push to trigger the workflow**

```bash
# Make a trivial change to frontend/ to trigger the path filter
echo "" >> frontend/content/blog/hello-world.md
git add .github/workflows/deploy-frontend.yml frontend/content/blog/hello-world.md
git commit -m "feat(cicd): frontend deploy workflow"
git push
```

- [ ] **Step 3: Watch the workflow**

Go to GitHub → Actions → "Deploy Frontend". Expected: green checkmark within ~3 minutes.

- [ ] **Step 4: Verify site is live**

```bash
curl http://$(cd infra && terraform output -raw lb_ip)/
```

Expected: HTML content of `index.html`.

## Task 3: Backend Deploy Workflow

- [ ] **Step 1: Create `.github/workflows/deploy-backend.yml`**

```yaml
name: Deploy Backend

on:
  push:
    branches: [main]
    paths:
      - "backend/**"
      - ".github/workflows/deploy-backend.yml"

permissions:
  contents: read
  id-token: write

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Authenticate to GCP
        uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: ${{ vars.WIF_PROVIDER }}
          service_account: ${{ vars.WIF_SERVICE_ACCOUNT }}

      - uses: google-github-actions/setup-gcloud@v2

      - name: Configure Docker for Artifact Registry
        run: gcloud auth configure-docker ${{ vars.GCP_REGION }}-docker.pkg.dev --quiet

      - name: Build and push image
        run: |
          IMAGE=${{ vars.BACKEND_IMAGE }}:${{ github.sha }}
          docker build -t $IMAGE backend/
          docker push $IMAGE
          docker tag $IMAGE ${{ vars.BACKEND_IMAGE }}:latest
          docker push ${{ vars.BACKEND_IMAGE }}:latest

      - name: Deploy to Cloud Run
        run: |
          gcloud run deploy backend \
            --image ${{ vars.BACKEND_IMAGE }}:${{ github.sha }} \
            --region ${{ vars.GCP_REGION }} \
            --project ${{ vars.GCP_PROJECT_ID }} \
            --quiet
```

- [ ] **Step 2: Commit and push**

```bash
# Trigger by touching a backend file
echo "# deployed" >> backend/app/routers/health.py
git add .github/workflows/deploy-backend.yml backend/app/routers/health.py
git commit -m "feat(cicd): backend deploy workflow"
git push
```

- [ ] **Step 3: Watch the workflow**

GitHub → Actions → "Deploy Backend". Expected: green within ~5 minutes (Docker build takes most of the time).

- [ ] **Step 4: Smoke test the deployed backend**

```bash
BACKEND_URL=$(cd infra && terraform output -raw backend_cloud_run_url)
curl "$BACKEND_URL/health"
```

Expected: `{"status":"ok"}`

## Task 4: Graph-Gen Job Deploy Workflow

- [ ] **Step 1: Create `.github/workflows/deploy-job.yml`**

```yaml
name: Deploy Graph-Gen Job

on:
  push:
    branches: [main]
    paths:
      - "jobs/graph-gen/**"
      - ".github/workflows/deploy-job.yml"

permissions:
  contents: read
  id-token: write

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Authenticate to GCP
        uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: ${{ vars.WIF_PROVIDER }}
          service_account: ${{ vars.WIF_SERVICE_ACCOUNT }}

      - uses: google-github-actions/setup-gcloud@v2

      - name: Configure Docker for Artifact Registry
        run: gcloud auth configure-docker ${{ vars.GCP_REGION }}-docker.pkg.dev --quiet

      - name: Build and push image
        run: |
          IMAGE=${{ vars.JOB_IMAGE }}:${{ github.sha }}
          docker build -t $IMAGE jobs/graph-gen/
          docker push $IMAGE
          docker tag $IMAGE ${{ vars.JOB_IMAGE }}:latest
          docker push ${{ vars.JOB_IMAGE }}:latest

      - name: Update Cloud Run Job image
        run: |
          gcloud run jobs update graph-gen \
            --image ${{ vars.JOB_IMAGE }}:${{ github.sha }} \
            --region ${{ vars.GCP_REGION }} \
            --project ${{ vars.GCP_PROJECT_ID }} \
            --quiet
```

- [ ] **Step 2: Commit and push**

```bash
git add .github/workflows/deploy-job.yml
git commit -m "feat(cicd): graph-gen job deploy workflow"
git push
```

- [ ] **Step 3: Watch the workflow**

GitHub → Actions → "Deploy Graph-Gen Job". Expected: green within ~5 minutes.

- [ ] **Step 4: Manually trigger the job to verify end-to-end**

```bash

## Task 5: End-to-End Smoke Test

LB_IP=$(cd infra && terraform output -raw lb_ip)

# Expected: HTTP/1.1 200, Content-Type: text/html

# API through CDN URL map

# Expected: {"status":"ok"}

curl http://$LB_IP/api/graph
# Expected: {"nodes":[...],"edges":[...]}
```

Note: HTTP will redirect to HTTPS once DNS and the managed cert are wired.

- [ ] **Step 2: Point DNS and verify TLS** (when ready)

In your DNS provider (Cloudflare or Route 53):
- Add an A record: `tapshalkar.com` → `LB_IP`
- Add a CNAME: `www.tapshalkar.com` → `tapshalkar.com`

Wait for managed cert provisioning (~15 min after DNS propagates):

```bash
gcloud compute ssl-certificates describe tapshalkar-cert --global
# status.domainStatus[].status should be "ACTIVE"
```

Then verify:
```bash
curl https://tapshalkar.com/health  # via CDN → Cloud Run
curl https://tapshalkar.com/        # via CDN → GCS
```
